Bootstrap Form Spam Filter

No CAPTCHA or Human Testers

Most spam emails will fill out all fields in a form based on the DOM elements on a page without reading the CSS. In this form there is a field called 'formChecker' wrapped in '.form-checker' with 'Display: none' added in the CSS. If this field is filled out a 'return false' is triggered in the JS, causing the email to not be sent.
Chrome, Safari, Firefox, IE9+

For IE8 support create the following script to display labels: 

<!--[if lt IE 8]>
<script>
$('#form-container label').css({'display':'inline-block !important'});
</script>
<![endif]-->

ASP initiate if formChecker is empty 

<%
    if (request("formChecker") = "") then
        ' all your form stuff here
    end if
%>

PHP initiate if formChecker is empty 

<?php
    if (request("formChecker") == "") {
        // all your form stuff here;
    }
?>
